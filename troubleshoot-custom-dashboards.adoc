= 疑難排解自訂儀表板
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
利用這些資訊解決在建立或使用自訂儀表板時可能遇到的常見問題。



[[dashboard-creation-failed]]
== 儀錶板建立失敗

**問題：**建立儀表板時，您會看到錯誤訊息「Failed to understand request and generate code」。

**原因：**系統無法理解您的要求或無法根據您的提示建立視覺化內容。

**解決方法：** 請嘗試以下方法：

* 請重新撰寫您的請求，更清楚說明您想要監控的內容。
* 從 link:create-custom-dashboard.html["建立自訂儀表板"] 中的其中一個範例提示開始。
* 將複雜的監控需求分解成幾個更簡單的儀表板。
* 指定要追蹤的工作負載、指標、時間範圍或維度。




[[dashboard-failed-to-load-data]]
== 儀錶板載入資料失敗

**問題：**儀表板已建立，但在載入資料時顯示錯誤。

**原因：**這通常是由於權限問題或資料來源無法使用造成的。

**解決方案：**

* **檢查權限：** 請確保您的 Workload Factory 帳戶擁有對儀表板使用的工作負載和資源的讀取存取。請查看 link:learn-custom-dashboards.html#aws-credentials-and-permissions["AWS 憑證和權限"] 中的權限要求。
* **驗證憑證：**確認您的 AWS 憑證有效且未過期。如需詳細資訊，請參閱 link:add-credentials.html["新增 AWS 認證"]。




[[dashboard-not-loading-or-loading-slowly]]
== 儀錶板無法載入或載入緩慢

**問題：**儀表板載入緩慢或某些小工具未顯示。

**原因：**載入速度慢可能是由於網路問題、資料量過大或臨時服務延遲造成的。

**解決方案：**

* **檢查您的網路連線：** 請確保您的網路連線穩定。
* **重新整理瀏覽器：** 請嘗試重新整理頁面，確認儀表板是否能正確載入。
* **清除瀏覽器快取和 cookie：** 這有助於解決因資料過期或損壞而導致的載入問題。
* **稍等片刻再試：**如果問題仍然存在，請稍等幾分鐘再試。




[[missing-dashboards]]
== 缺少儀表板

**問題：**您建立的儀表板不再列在儀表板目錄中。

**原因：**您 Workload Factory 帳戶中的其他使用者可能已將其刪除。

**解決方案：**

* 請確保您已登入正確的 Workload Factory 帳戶。
* 詢問您帳戶中的其他使用者是否有人刪除了控制面板。
* 如有必要，請根據原始要求重新建立儀表板。


Workload Factory 帳戶中的所有使用者都可以檢視和刪除自訂儀表板。管理共享儀表板時，請與您的團隊協調。
