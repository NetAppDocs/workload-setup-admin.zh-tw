= 了解自訂儀表板
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
身為雲端和儲存管理員，自訂儀表板能讓您清楚了解所管理的系統和工作負載。您可以根據自身的工作、優先順序和業務目標建立和個人化檢視，而無需局限於預設版面配置。

自訂儀錶板利用 AI 技術，建立個人化的 Workload Factory 環境視圖。您只需用簡單易懂的語言描述想要監控的內容，系統即可根據您的需求自動產生視覺化圖表。

Workload Factory 帳戶中的所有使用者都可以查看所有自訂儀表板。這種共享設定有助於團隊協作，並在整個組織內查看相同的營運資訊。



[[why-use-custom-dashboards]]
== 為什麼要使用自訂儀錶板

Workload Factory 使用者需要管理大量的工作負載、環境和優先順序，但標準儀表板僅顯示預設視圖。自訂儀表板有助於解決以下幾個運維難題：

* *快速回答關鍵問題*：專注於最重要的問題或理解上的不足，而不是一般的系統統計數據。
* *Monitor what matters now*：追蹤與您目前優先事項相符的指標，而不是一組固定的健康指標。
* *將技術資料與業務成果連結*：展示系統效能如何影響營運和業務目標。
* *避免在工具之間切換*：在一個地方查看所有相關資訊－無需在工具、頁面或匯出之間切換。




[[what-you-can-do-with-custom-dashboards]]
== 自訂儀表板的功能

在設定過程中，人工智慧引導的對話框可協助您定義：

* 需要監控的工作負載和資源
* 對您的工作至關重要的關鍵指標
* 要比較的時間範圍和時期
* 按地區、工作負載或其他類別查看
* 您需要追蹤和展示的規則或要求




[[considerations-and-limitations]]
== 考慮因素和局限性



[[aws-credentials-and-permissions]]
=== AWS 憑證和權限

若要建立或檢視自訂儀表板，您需要擁有要監控的資源的正確 AWS IAM 權限。請確保 Workload Factory 帳戶對儀表板使用的資料來源具有讀取存取權限。

為了建立儀表板，Workload Factory 使用唯讀的 AWS 憑證。這些憑證僅允許對下列服務執行讀取操作，例如 `fsx:DescribeVolumes`和 `fsx:DescribeFileSystems`：EC2、FSx、CloudWatch、Resource Groups Tagging API、備份、CloudWatch Logs、KMS 和 EFS。

link:add-credentials.html["了解如何新增 AWS 憑證並設定 IAM 權限"^].



[[link-to-fsx-for-ontap]]
=== 連結至 FSx for ONTAP

Workload Factory 需要 Link 才能連接到 FSx for ONTAP 檔案系統。Link 在 Workload Factory 與一個或多個 FSx for ONTAP 檔案系統之間建立信任關係和連線。這可讓您直接透過 ONTAP REST API 呼叫來監控和管理某些 Amazon FSx for ONTAP API 無法提供的檔案系統功能。

https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["學習如何建立連結"^].



[[read-only-design]]
=== 唯讀設計

自訂儀表板為唯讀。無法變更您的儲存或工作負載設定。這可讓您監控系統，而不會意外變更任何內容。



[[limitations]]
=== 限制

目前自訂儀表板有以下限制：

* 您無法在 Workload Factory 帳戶之外共用儀表板




[[demo-environment]]
=== 示範環境

如果您使用的是 Workload Factory 示範帳戶，則可以使用一些預建的儀表板。這些儀表板可協助您了解自訂儀表板的功能。每個儀表板都展示了一個常見的監控需求：災難復原狀態、跨區域的容量和成本，以及是否遵循儲存保護規則。



[[sample-prompts]]
=== 範例提示

以下範例展示如何用簡單易懂的語言描述您的監控需求：

* *災難復原監控*：「我正在 AWS 上為我的本地系統設定災難復原解決方案。我希望看到我的本地資源的完整清單，以及每個資源在美國東部區域災難復原站點的狀態，包括是否已設定、是否處於作用中狀態以及是否運作正常。」
* *多區域容量和成本*：「我管理跨多個雲端區域和應用程式的儲存空間。我希望清楚了解已分配的儲存量、已使用的儲存量及其成本，並能查看每個區域和每個應用程式的詳細資訊。」
* *儲存保護合規性*：「我正在根據組織的標準建立新的儲存保護策略。請向我展示所有檔案系統，並說明它們是否符合這些要求：
+
** 資料庫工作負載：每個磁碟區每天 12 個快照，每週 2 個防篡改快照。
** NFS 和 SMB 共用：每個磁碟區每天 7 個快照，每週 1 個防竄改快照。
** 所有磁碟區都必須啟用 ARP。"
